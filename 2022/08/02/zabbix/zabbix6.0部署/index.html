基础信息

操作系统/安装软件 版本
ubuntu 20.04.1
kernel 5.4.0-122-generic
mysql 8.0.30
zabbix_server、zabbix_agent 6.0.7

安装zabbix

安装zabbix存储库

# wget https://repo.zabbix.com/zabbix/6.0/ubuntu/pool/main/z/zabbix-release/zabbix-release_6.0-3+ubuntu20.04_all.deb
# dpkg -i zabbix-release_6.0-3+ubuntu20.04_all.deb
# apt update

安装zabbix server, frontend, agent

# apt install zabbix-server-mysql zabbix-frontend-php zabbix-nginx-conf zabbix-sql-scripts zabbix-agent2

初始化数据库

安装要求：

数据库软件 支持的版本
MySQL/Percona 8.0.X 
# mysql -uroot -h 127.0.0.1 -p
password
mysql> create database zabbix character set utf8mb4 collate utf8mb4_bin;
mysql> create user 'zabbix'@'%' identified by 'password';
mysql> grant all privileges on zabbix.* to 'zabbix'@'%';
mysql> flush privileges;
mysql> quit;

导入初始化数据：

# zcat /usr/share/doc/zabbix-sql-scripts/mysql/server.sql.gz | mysql -uzabbix -h 127.0.0.1 -p -D zabbix

配置zabbix server的数据库连接密码

编辑/etc/zabbix/zabbix_server.conf文件：

DBHost=127.0.0.1
DBUser=zabbix
DBPassword=password

配置zabbix的前端

编辑/etc/zabbix/nginx.conf文件，取消对listen和server_name指令的注释：

listen          8080;
server_name     192.168.168.226;

启动zabbix服务和agent

# systemctl restart zabbix-server zabbix-agent2 nginx php7.4-fpm
# systemctl enable zabbix-server zabbix-agent2 nginx php7.4-fpm

配置zabbix fronend

访问http://192.168.168.226:8080/setup.php

汉化

# 安装中文库
$ apt install language-pack-zh-hans -y

重启server：systemctl restart zabbix-server，打开http://192.168.168.226:8080/setup.php页面，查看Default language是否可选Chinese（zh-CN）。如果不行，请按照以下方式继续：

编辑/etc/locale.gen文件

# 取消注释
zh_CN.UTF-8 UTF-8

编译语言包

$ locale-gen
Generating locales (this might take a while)...
  en_US.UTF-8... done
  zh_CN.UTF-8... done
  zh_SG.UTF-8... done
Generation complete.

此时清空缓存重新打开http://192.168.168.226:8080/setup.php页面，查看Default language是否可选Chinese（zh-CN）。

如若不可以：

编辑/etc/default/locale文件，修改系统默认语言：

LANG=en_US.UTF-8
改为
LANG=zh_CN.UTF-8

修改之后重启服务器。

设置流程

登录

zabbix默认用户名：Admin，默认密码：zabbix

解决中文乱码

默认安装的zabbix显示中文字体时存在乱码情况，打开监测->主机->图形可发现中文字体无法正常显示：

zabbix默认的字体文件为：/usr/share/zabbix/assets/fonts/graphfont.ttf -> /etc/alternatives/zabbix-frontend-font，可通过使用windows上的字体文件进行替换修复此问题。

步骤：

  1. win+r输入fonts，选择自己喜欢的字体
  2. 这里我选择华文楷体 常规上传至服务器
  3. 覆盖原有的字体文件：cp -r -p /home/guptwlxy/STKAITI.TTF /etc/alternatives/zabbix-frontend-font
  4. 刷新zabbix页面即可

zabbix agent & agent2介绍

https://www.zabbix.com/documentation/6.0/zh/manual/appendix/agent_comparison

zabbix agent工具用于采集主机的数据并上报至zabbix server，部署于需要监控的主机节点，目前有zabbix-agent和zabbix-agent2两种，简略对比情况如下：

参数 zabbix-agent zabbix-agent2
编程语言 C Go，部分使用C
守护进程 支持 仅由 systemd （在 Windows 上是）
支持的扩展 C中的自定义可加载模块 Go 中的自定义插件
进程 每个服务器/代理记录的单独主动检查进程。 具有自动创建线程的单个进程。
最大线程数由 GOMAXPROCS 环境变量确定。
指标 UNIX: 查看支持的监控项列表。
Windows: 查看其他Windows 特定监控项的列表。		 UNIX: Zabbix agent支持的所有指标。
此外，代理 2 为以下对象提供 Zabbix-native 监控解决方案：Docker, Memcached, MySQL, PostgreSQL, Redis, systemd,和其他监控目标 - 请参阅代理 2 特定监控项的完整列表。
Windows: Zabbix agent代理支持的所有指标，以及HTTPS、LDAP 的net.tcp.service* 检查。
此外，agent 2 为 PostgreSQL、Redis 提供了 Zabbix-native 监控解决方案。
并发 单个服务器的主动检查按顺序执行。 来自不同插件的检查或一个插件内的多个检查可以同时执行。
持久性存储 不支持 支持

关键点：zabbix-agent2不仅支持zabbix-agent的监控项，还支持对Docker, Memcached, MySQL, PostgreSQL, Redis, systemd等进行监控，所以一般推荐使用zabbix-agent2，官方默认安装的zabbix-agent。

批量安装（linux）

# 下载并安装zabbix-agent2
$ ansible linux -m shell -a 'wget https://repo.huaweicloud.com/zabbix/zabbix/6.0/rhel/7/x86_64/zabbix-agent2-6.0.7-1.el7.x86_64.rpm -O /tmp/zabbix-agent2-6.0.7-1.el7.x86_64.rpm && yum localinstall /tmp/zabbix-agent2-6.0.7-1.el7.x86_64.rpm -y && rm -fr /tmp/zabbix-agent2-6.0.7-1.el7.x86_64.rpm'

# 修改配置文件(192.168.168.226为zabbix server主机IP)
$ ansible linux -m shell -a 'sed -i "s#Server=127.0.0.1#Server=192.168.168.226#g" /etc/zabbix/zabbix_agent2.conf'

# 启动服务
$ ansible linux -m systemd -a 'name=zabbix-agent2.service state=started enabled=true'

# 添加防火墙策略
$ ansible linux -m shell -a 'iptables -I INPUT -p tcp -s 192.168.168.226 --dport 10050 -j ACCEPT && service iptables save'

批量安装（windows）

# 下载zabbix-agent2
$ wget https://cdn.zabbix.com/zabbix/binaries/stable/6.0/6.0.7/zabbix_agent2-6.0.7-windows-amd64-openssl.msi

# 创建安装脚本
$ cat install.bat
@echo off
title install zabbix-agent2 
:: 获取以太网ip
for /f "tokens=16" %%i in ('ipconfig ^|find /i "ipv4"') do set ip=%%i
:: 安装zabbix_agent2-6.0.7-windows-amd64-openssl.msi
msiexec /i C:\Users\Administrator\Desktop\zabbix_agent2-6.0.7-windows-amd64-openssl.msi /qb InstallFolder="E:\bestmem\bestmem_install\Zabbix Agent 2\"  Server="192.168.168.226"  Hostname=%ip%

# 拷贝安装包和安装脚本到目标主机
$ ansible windows -m win_copy -a 'src=./zabbix_agent2-6.0.7-windows-amd64-openssl.msi dest=C:\Users\Administrator\Desktop\'
$ ansible windows -m win_copy -a 'src=./install.bat dest=C:\Users\Administrator\Desktop\'
$ ansible windows -m win_shell -a 'C:\Users\Administrator\Desktop\install.bat'

批量添加监控

首先在配置–>主机–>创建主机中创建一个主机模板：

主机名称应与可见的名称一致，否则无法批量修改。

批量添加的主机应该确保模板、群组的信息是一致的，不一致的部分后续手动修改。

然后将该模板导出：

将zbx_export_hosts.yaml文件上传至linux服务器，并在同级目录下创建以下shell脚本，执行后生成批量导入文件zbx_input_hosts.yaml：

修改变量zbx_export_hosts为主机模板文件zbx_export_hosts.yaml中的主机名称字段

修改变量hosts为需要批量添加的主机

[root@194 all]# ls
zbx_export_hosts.yaml  zbx_input_hosts.sh  zbx_input_hosts.yaml

[root@194 all]# cat zbx_input_hosts.sh
#!/bin/bash
zbx_export_hosts=192.168.168.192	# 修改为主机模板文件zbx_export_hosts.yaml中的主机名称字段
hosts=(192.168.168.202 192.168.168.206 192.168.168.208 192.168.168.209 192.168.168.218 192.168.168.219 192.168.168.220 192.168.168.221 192.168.168.222 192.168.168.225)	# 需要批量添加的主机
tmp_name=`pwd`/zbx_export_hosts.yaml
if [ ! -f ${tmp_name} ]; then
   echo '没有zabbix模板文件: zbx_export_hosts.yaml'
   exit 1
fi
for host in ${hosts[@]}; do
    if [ $host == ${hosts[0]} ]; then
        sed "s#${zbx_export_hosts}#${host}#g" ${tmp_name} >> zbx_input_hosts.yaml
        continue
    fi
    sed -n "s#${zbx_export_hosts}#${host}#g;/hosts:/,\$p" ${tmp_name} | tail -n +2 >> zbx_input_hosts.yaml
done

[root@194 all]# bash ./zbx_input_hosts.sh

执行成功后，拷贝zbx_input_hosts.yaml文件到桌面，并进行导入：

等待片刻，可查看可用性是否正常：

灰色为初始状态，绿色为正常状态，红色为警告状态

如果一直为灰色，可通过重启zabbix server来解决。如果为红色，需要检查zabbix agent是否正常。
