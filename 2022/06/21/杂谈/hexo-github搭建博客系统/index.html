创建github仓库

安装nodejs

官网下载安装包，安装即可

$ node -v
v16.14.2
$ npm -v
8.5.5

安装hexo-cli

# 使用 npm 全局安装 hexo-cli
$ npm install hexo-cli -g --registry=https://registry.npm.taobao.org

# 安装完成后本地初始化站点目录
$ hexo init bestmem.github.io
$ cd bestmem.github.io
$ npm install --registry=https://registry.npm.taobao.org

插件安装

# 该插件用于hexo上传静态文件至github仓库，--save的作用为将该插件记录到package.json文件中，方便后续npm install直接安装。
npm install hexo-deployer-git --save

目录结构介绍

$ tree -L 1
.
├── _config.landscape.yml
├── _config.yml					# 网站的配置信息
├── node_modules			
├── package.json				# 应用程序信息	
├── package-lock.json
├── scaffolds					# 模版文件夹
├── source						# 资源文件夹是存放用户资源的地方
└── themes						# 主题文件夹。Hexo 会根据主题来生成静态页面。

执行hexo init的时候，会从github上clone相关代码，若warning SSL证书相关的问题，可通过git config --global http.sslVerify false忽略证书的验证。

基础配置

配置文件分为网站配置文件和主题配置文件

  • 网站配置文件位于：bestmem.github.io/_config.yml
  • 主题配置文件位于：themes目录下相应的主题内的_config.yml

修改网站配置文件

https://hexo.io/zh-cn/docs/configuration

$ vim _config.yml
# Site
title: Hexo						# 设置title
subtitle: ''
description: ''
keywords:
author: bestmem					# 设置作者信息
language: zh-CN					# 设置语言
timezone: 'Asia/Shanghai'		# 设置时区

# 配置你的网站站点地址
url: https://bestmem.github.io/

# 配置本地静态文件需要部署的仓库地址
deploy:
  type: 'git'
  repo: https://github.com/bestmem/bestmem.github.io.git
  branch: master

现在我们可以尝试在本地运行默认配置的博客系统

$ hexo server
INFO  Validating config
INFO  Start processing
INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

访问http://localhost:4000

运行没有问题后，可通过如下命令生成静态文件

$ hexo generate

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令hexo clean，清除缓存文件 (db.json) 和已生成的静态文件 (public)。然后再重新生成静态页面。

并将其部署到github服务器

$ hexo deploy

随后便可打开你的github站点地址进行访问：https://bestmem.github.io/

其它配置

图片加载

外部存储

可以使用Typora+PicGo+aliyun OSS将图片保存在对象存储中，方便文章加载图片。

本地存储

修改主配置文件_config.yml，开启自动创建与文章同级同名的图片存储目录

post_asset_folder: true

安装插件：

npm i https://github.com/7ym0n/hexo-asset-image --save

注：使用https://github.com/7ym0n/hexo-asset-image插件，而不是hexo-asset-image插件，否则可能会造成生成静态文件后网页加载图片路径错误。

创建文章：

$ hexo new 本地存储使用文章
INFO  Validating config
INFO  Created: C:\bestmem\document\bestmem.github.io\source\_posts\本地存储使用文章.md

打开文件夹C:\bestmem\document\bestmem.github.io\source\_posts\，会发现多了名为本地存储使用文章的文件夹，用于存储图片文件。

在文章本地存储使用文章.md中通过相对路径加载图片

![](本地存储使用文章/测试.png)

生成静态文件并推送至github：

$ hexo clean && hexo g
$ hexo d

文章分类存储

默认我们使用hexo new [layout] (title)创建文章时，是在source/_posts下生成的文章Markdown文件和图片存储目录，这会导致当我们有大量文章时，该目录显得杂乱。

因此好的规范是：根据你博文的分类categories来创建相应的父目录存储文章。

我们的hexo-github搭建博客系统.md属于杂谈分类，因此我们的创建方式如下

$ hexo new post -p 杂谈/hexo-github搭建博客系统

命令使用

新建一个网站

$ hexo init [folder]

创建文章

$ hexo new "hexo+github搭建博客系统"
INFO  Validating config
INFO  Created: C:\bestmem\document\bestmem.github.io\source\_posts\hexo-github搭建博客系统.md

直接编写该Markdown文件即可开始你的博客之旅了。

本地访问

$ hexo server

清除缓存文件

$ hexo clean

生成静态文件

$ hexo generate		# 可简写为hexo g

部署服务

$  hexo deploy

列出网站资料

$ hexo list <type>
Arguments:
  type  Available types: page, post, route, tag, category

发表草稿

$ hexo publish [layout] <filename>

hexo默认提供了主题库：点我访问，我们可以选择自己喜欢的主题，在github上查找其下载地址。也可以通过github搜索语法自行搜索热门主题hexo theme stars:>1000

插件安装

npm i hexo-generator-feed --save		# 添加 RSS 订阅支持

hexo new page 404			# 新建 404 页
hexo new page "friends"		# 新建友情链接 friends 页

hexo的快速使用

创建一个新帖子

hexo new "hexo+github搭建博客系统"

本地运行服务

hexo server

生成静态文件

hexo generate

部署静态文件到远程github站点

hexo deploy
